lao documentation

lao version 1.1.0 - 20170904

info

A table describing a driver.

  • type
    • Either "live" or "file" depending on if the driver is a live-output or a file-output driver.
  • name
    • A longer name for the driver which may contain whitespace, but no newlines. It is useful for telling users what output driver is in use.
  • shortName
    • A short identifier for the driver. This contains only alphanumeric characters and no whitespace. It is used to look up the driver ID number with ao.driverId.
  • preferredByteFormat
    • Specifies the preferred ordering of bytes. Can be "native", "big" or "little".
  • priority
    • A positive number ranking how likely it is for the driver to be used as the default. The higher, the better.
  • comment
    • A driver comment string. May contain newlines.
  • options
    • A table listing the options available for this driver.

Notes

Instead of this being a userdata which is restrictive and cumbersome, you use a regular table with a special set of members.
