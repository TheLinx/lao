lao version 1.0.0 - 20100905

ao.openLive

Open a live playback device for output. 

userdata device, string error = ao.openLive(number driverId, table format, table options)

Parameters

  • driverId
    • The ID number of the driver as returned by either ao.driverId or ao.defaultDriverId.
  • format
    • A table describing the sample format.
    • See also: sampleFormat table
  • options
    • A table containing options.
    • See also: options table

Return Values

  • device
    • A non-nil result indicates success. You can now use the devices metamethods play and close to play audio and close the device.
    • A nil result means something went wrong. The error message is returned as the second return value.
  • error
    • A string describing what went wrong (if anything went wrong). This can be one of the following:
      • no such driver
      • not a live-type driver
      • a valid option-key has an invalid value
      • cannot open the device
      • something went wrong

Notes

File output drivers cannot be used with this function. Use ao.openFile instead.
